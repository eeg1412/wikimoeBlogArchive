emlog模板《black memory v2》发布

博文 作者：广树 时间：2018-09-04 21:35 分类：PHP

这套模板其实就是我现在所使用的emlog模板。基于emlog模板《black_memory》进行了大幅改动。



主要增加和修改了如下内容：

1.模板由原先的仅支持PC端改为了响应式模板。

2.修改了侧边栏的分类与归档的样式。

3.增加了背景动画效果。

4.幻灯片由原先的最多三张改为了最多十张。

5.评论由多级嵌套改模式为了两层模式。

6.增加了一个追番页面。<预览>

7.增加了一个抽卡页面。<预览>

8.增加了一个命运页面。<预览>

9.增加了文章代码显示效果。

10.评论追加了表情功能。

11.评论追加了查看卡牌功能。

12.修复了大量的CSS问题和JS问题。

PS.不带关于页面、赞助页面、友链页面、广告位，因为这些都是我自用的。

PS2.模板和插件基于emlog 5.3.1开发，其他版本有没有问题未知。

模板使用方法：

1.安装模板设置插件。

2.安装模板。

3.启动模板。

4.在当前模板中就会多出一个设置按钮，在设置中设置所需要的功能。


模板设置：

网站LOGO设置：

设置的LOGO会显示在网站导航栏的左上角，上传自己心仪的LOGO即可。


bangumi账号与bangumi密码：

用于追番页面显示自己追番信息的bangumi账号密码。需要在bangumi注册账号。并设置自己的追番信息。如果不使用追番页面的话则无需填写。


抽卡系统路径：

用于在评论栏出现 TA的卡牌 字样。引导访客查看评论者的卡牌收集情况，需要安装上面提到的《维基萌抽卡系统》。为空则为不显示。

注意：这里填写的路径是创建抽卡页面之后，减去站点地址的路径。

注意【?】和【&】的使用

例子：

我这里抽卡地址是https://www.wikimoe.com/?post=130，我的站点地址是https://www.wikimoe.com/，则这里填写【?post=130&】。

如果你的站点地址是https://www.host.com/blog/，抽卡页面是https://www.host.com/blog/?post=130，则也只用填写【?post=130&】。

如果你的站点地址是https://www.host.com/，抽卡页面是https://www.host.com/post/130，则这里需要填写【post/130?】（因为这里前面没有【?】，所以【&】就变成了【?】）。


幻灯片：

可选择是否开启，下面的幻灯片图片和地址按需上传和填写。

注意：会覆盖原先的图片！


关于如何创建页面：

进入emlog后台，点击页面，点击新建页面，在【发布页面】按钮上方有一个【页面模板】。在这里填写不同页面的名字就会对应不同的页面模板。

bangumi -> 追番页

card -> 抽卡页

programmerCalendar -> 命运页
